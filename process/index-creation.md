Below is a description of the process used to create the Federal Website Index, which is then used as the target URL list for the Site Scanning engine to scan. The actual code that does this is [here](https://github.com/GSA/federal-website-index/blob/main/builder/src/main.ts).   

* The Federal Website Index is created by combining and processing a number of [individual source datasets](https://github.com/GSA/federal-website-index/tree/main/data/source-descriptions#readme).  The list of datasets is managed [here](https://github.com/GSA/federal-website-index/blob/main/builder/src/main.ts#L50-L77) (in the `fetchAllSourceListData` function), and the urls for these datasets are managed [here](https://github.com/GSA/federal-website-index/blob/main/builder/src/config/source-list.config.ts).  
* The specified source datasets are copied and imported into memory. In the process, each URL is cleaned by removing any protocols, `www.`, and paths.  Snapshots of each individual dataset are stored [here](https://github.com/GSA/federal-website-index/tree/main/data/source-list-snapshots).
* The various source datasets are combined.  A snapshot of this combined list is stored [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-union.csv).  
* The combined list of websites is then deduplicated.  A snapshot of the dedupped list is stored [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-dedup.csv).
* The base domain and top level domain fields are extrapolated and added as new fields, a snapshot of which is [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-add-base_domain-tld.csv).  
* Agency, bureau, and branch information is added to each website by pulling in the relevant information for its base domain from the [these](https://github.com/GSA/federal-website-index/blob/main/data/source-descriptions/dotgov-registry-federal.md) [three](https://github.com/GSA/federal-website-index/blob/main/data/source-descriptions/dotmil_domains.md) [files](https://github.com/GSA/federal-website-index/blob/main/data/source-descriptions/non_dotgovdotmil_domains.md) (snapshots of which are [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-GOV-agency-bureau-merge.csv), [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-MIL-agency-bureau-merge.csv), and [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-OTHER-FEDERAL-agency-bureau-merge.csv)).
* Agency and bureau information in the [list of websites from OMB's 21st Century IDEA engagement](https://github.com/GSA/federal-website-index/blob/main/data/source-descriptions/omb_idea.md) are then used to override the information from the previous step since it should be more accurate.  A snapshot of the end result is [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-OMB-agency-bureau-merge.csv).  
* The list of websites is then compared to two ignore files (the [`begins with` list](https://github.com/GSA/federal-website-index/blob/main/criteria/ignore-list-begins.csv) and [`contains` list](https://github.com/GSA/federal-website-index/blob/main/criteria/ignore-list-contains.csv)) [note that the `contains` list actually requires that the specified string have non-alphanumeric characters both before and after it (or just after, if the string is at the beginning)]. The purpose of this is to try and identify non-public websites which are then labeled `Filtered = TRUE`.
* The same [three](https://github.com/GSA/federal-website-index/blob/main/data/source-descriptions/dotgov-registry-federal.md) [domain](https://github.com/GSA/federal-website-index/blob/main/data/source-descriptions/dotmil_domains.md) lists above are then used to limit the list of websites to those with currently-registered, federal domains.  The result is snapshotted [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-gov_mil-filter.csv), and the list of sites that are removed can be found [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-dead-sites-filter-removed.csv).   
* Analytics data is then added in by pulling in pageviews and visits from [this copy](https://github.com/GSA/federal-website-index/blob/main/data/source-lists/dap_top_100000_domains_30_days.csv) of [this dataset](https://analytics.usa.gov/data/live/top-100000-domains-30-days.csv).  A snapshot is then stored [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-dap_top_100000_merge.csv).
* The dataset's columns are then reordered to align with what the Site Scanning end data looks like.  A snapshot is [here](https://github.com/GSA/federal-website-index/blob/main/data/process-snapshots/after-column-reorder.csv).
* As a final step, the dataset is alphabetized by `base_domain` then `initial_url`.  The result is stored [here](https://github.com/GSA/federal-website-index/blob/main/data/site-scanning-target-url-list.csv) as the completed website index.
